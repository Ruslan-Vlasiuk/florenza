# Florenza — Admin Guide для Варвари

> **Цей документ — путівник по адмінці Florenza.** Адмінка живе за адресою `florenza-irpin.com/admin` (логін і пароль будуть надані після розгортання).

---

## Перші кроки після розгортання

1. **Заходимо в адмінку**: `florenza-irpin.com/admin`
2. **Логін:** твій email + пароль (надається підрядником, потім ти його змінюєш)
3. **Перший раз** — обов'язково:
   - Зміни пароль на свій
   - Включи 2FA (Google Authenticator)
   - Запиши resque-коди в безпечне місце

## Огляд розділів

### 🎛 Дашборд

**Що бачиш одразу при вході:**

- **Нові заказы** (потребують уваги) — червоні картки
- **Сьогоднішні доставки** — слот за слотом, з адресами
- **Ескалації** — діалоги де Лія передала тобі управління
- **Утренній дайджест** — *«За ніч 4 заказа на 12 800 грн, всі автоматично оплачені, 1 потребує уваги»*
- **Метрики:** трафік сайту, кількість діалогів, конверсія в заказ
- **AI-витрати:** скільки витрачено на Claude і Gemini за поточний місяць

### 🌸 Букети

**Список усіх букетів** з фільтрами по типу, квітці, приводу, статусу.

**Створення / редагування букета:**

| Поле | Що робити |
|---|---|
| Назва | Пишеш сама або кнопка 🤖 «Запропонувати назву» — Лія дає 3 варіанти |
| Опис короткий | Пишеш сама або 🤖 «Згенерувати опис» |
| Опис розгорнутий | Те саме |
| Склад | Структуроване поле: квітка + кількість + сорт. Кнопка 🤖 «Підказати склад» |
| Ціна | Ручне поле |
| Знижка | Розкривається секція: % або фікс. сума, дата початку, дата кінця, назва акції |
| Тип / квітка / привід | Multi-select |
| Розмір | S/M/L + висота/діаметр у см |
| Сезонність | «цілорічно» або діапазон місяців |
| **Студія фото** | Окрема секція — два режими: «Згенерувати з нуля» / «Перенести у наш стиль» |
| Галерея | Drag-n-drop додаткові фото |
| SEO-теги | 🤖 кнопки для meta_title, meta_description, alt-теги |
| Статус | Draft / Published / Hidden / Sold out |

**Bulk-операції:**
- Виділення кількох букетів → дії: «Створити акцію», «Скасувати акцію», «Перенести в категорію», «Видалити»

### 📦 Замовлення

**Канбан-вид** з drag-n-drop статусів:

```
Новий → Очікує оплати → Оплачений → У роботі → Передано курʼєру → Доставлено
                                ↓
                            Скасовано
```

**Картка заказу:**
- Усі дані замовника, отримувача, адреси
- Склад букета, ціна, знижка
- Метод оплати, статус оплати, чек ПРРО (посилання)
- Привʼязаний діалог з Лією (вкладка «Розмова»)
- Кнопки: «Створити чек ПРРО», «Перейти до доставки», «Скасувати», «Повідомити клієнта»

**Дії за статусом:**
- *Новий* → Лія обробляє автоматично
- *Очікує оплати* → можна вручну надіслати нагадування або скасувати
- *Оплачений* → почни збирати букет
- *У роботі* → коли почав збирати
- *Передано курʼєру* → коли вийшов на доставку
- *Доставлено* → статус автоматично через час або вручну (Лія через 4 години запитає клієнта)

### 💬 Інбокс

**Усі діалоги** з усіх каналів (сайт-чат, Telegram, Viber) в одному місці.

**Фільтри:**
- Канал
- Статус: Активний / Ескалований / Закритий / Заказ оформлено
- Тег: wedding / corporate / complaint / vip / funeral_sensitive
- Період

**Картка діалогу:**
- Усі повідомлення (Лія + клієнт + ти)
- Контекст клієнта: телефон, історія заказів, преференції, статус знижки
- Кнопка «Підключитись» — пишеш від імені «Лія», клієнт не помічає переходу. Опційно — натискай **«Розкрити що це я особисто»** і клієнту прийде system message *«Тепер з вами Варвара особисто»*

**Кнопка «Покращити промпт»** на повідомленні Лії:
- Виділяєш місце де вона помилилась
- Пишеш «правильна відповідь має бути така»
- Ця інформація автоматично потрапляє в розділ «Лія — правила» → FAQ
- Лія наступного разу відповість правильно

### 📸 Студія фото

**Окремий розділ** для генерації і редагування фото.

**Режим 1: «Згенерувати з нуля»**
- Опис букета (буде підмішаний до глобального стилевого промпта)
- Кнопка «Створити 4 варіанти» → за 5–15 секунд показує 4 фото
- Обираєш найкраще → утверджуєш як основне
- Решту можна зберегти в галерею

**Режим 2: «Перенести у наш стиль»**
- Drag-n-drop твоє фото (з телефона, від друга, з референсу)
- Опис що зберегти, що змінити (опційно)
- Кнопка «Перенести» → 4 варіанти
- Обираєш найкраще

**Налаштування глобального стилю:**
- Розділ «Налаштування бренду» → «AI-фото стиль»
- Можна редагувати глобальний промпт (наприклад змінити «cream background» на «aged plaster»)
- Версіонується автоматично, можна повертатися до старіших версій
- **Перші 1–2 тижні після запуску — період калібровки.** Не бійся експериментувати

### 📰 Журнал/Блог

**Два режими роботи:**

**Автогенерація (default):**
- Налаштування «Скільки статей на день» (0–5)
- Чергa тем — або автоматична (на основі сезонності + Google Trends), або ручна
- Розклад публікації (наприклад, 09:00 щодня)
- Перемикач «Автопублікація / Спочатку в драфт на ревью»

**Ручний контроль:**
- Список усіх статей з статусами
- Топ-10 статей помічені як «потребує ручної доводки» — переглядай їх перед публікацією
- Кнопка «Створити нову статтю» — пишеш тему, AI робить драфт, ти редагуєш

**Метрики:**
- Покази / кліки / середня позиція в Google
- Статті з низьким CTR — підсвічуються червоним, AI пропонує апдейт

### 🎤 Голос бренду

**Структурований редактор** (НЕ markdown, форма з полями):

| Розділ | Що тут |
|---|---|
| Тон | Слайдери: формальність (1–10), теплота (1–10), поетичність (1–10) |
| Принципи | Список тверджень: «не використовуємо уменьшительные», «звертаємось на Ви», «не обіцяємо більше ніж можемо» |

**Все це компілюється в системний промпт** для Лії та для AI-content генерації. Зміни → одразу впливають на всі майбутні AI-відповіді.

### 🤖 Лія — правила

**Структурований редактор поведінки AI-Концʼєржа:**

| Розділ | Що тут |
|---|---|
| Представлення | Текст з яким Лія представляється при першому повідомленні |
| FAQ | Список типових питань з еталонними відповідями (Лія використовує їх дослівно або адаптує) |
| Що зараз НЕМАЄ | Ad-hoc поле: «сьогодні немає півоній», «не приймаю на 1000 троянд» — редагується щодня |
| Зони доставки | Підгружується автоматично з розділу «Зони» |
| Способи оплати | Підгружується з розділу «Платежі» |
| Цінова політика | Що Лія може обіцяти, які знижки, які ні |
| Тригери ескалації | Слова/контексти при яких Лія обовʼязково передає тобі: «весілля», «корпоратив», «похорон», «скарга» |
| Заборонені теми | Що Лія не обговорює (політика, особисті думки) |
| Відмовлення з повагою | Шаблони відмов — наприклад для дальніх замовлень за межі ИБГ + Києва |

### 💸 Знижки

**Список усіх активних і запланованих знижок.**

**Створення знижки:**
- Обираєш букет(и) — окремо або bulk
- % або фікс. сума
- Період: дата+час початку → дата+час кінця
- Назва кампанії (опційно): «8 березня», «Розпродаж сезонних»

**Заплановані знижки** — створюєш зараз, активуються в потрібну дату.

**Деактивація:** одна кнопка вимикає акцію миттєво.

**Аналітика:**
- Скільки замовлень куплено зі знижкою
- Скільки разів відображено зі знижкою
- ROI акції

### 📨 Розсилки

**Створення розсилки:**

1. Заголовок (внутрішній)
2. Текст — пишеш сама або 🤖 «Запропонуй текст для 8 березня»
3. Сегментація:
   - Усі клієнти
   - Купували в останні N місяців
   - Купували на конкретний привід в минулому році
   - VIP (сума > X грн)
   - Канал: тільки Telegram / тільки Viber / обидва
4. Розклад: зараз / на дату / повторювана
5. Передперегляд: «Ця розсилка піде 47 людям» + тестове повідомлення тобі
6. Запустити

**Захист:**
- Frequency cap: 1 розсилка на клієнта на 30 днів (hard)
- Annual cap: 4 розсилки на клієнта на рік (hard)
- Якщо клієнт натиснув `/stop` — автоматично виключається назавжди

**Аналітика рассылки:**
- Доставлено / прочитано / відписалось / замовлень за 7 днів

## Часті питання

### «Як змінити ціну букета?»
Букети → знайди потрібний → поле «Ціна» → змінив → Save. Сайт оновиться миттєво.

### «Як швидко відповісти клієнту замість Лії?»
Інбокс → знайди діалог → кнопка «Підключитись» → пиши. Клієнт бачить ім'я «Лія», переключення непомітне. Якщо хочеш щоб клієнт зрозумів що це ти — натисни «Розкрити».

### «Як вимкнути прийом замовлень на сьогодні?»
Графік → перемикач «Прийом замовлень на сьогодні: ON/OFF» → OFF.

### «Як побачити витрати на AI?»
Аналітика → вкладка «Витрати». Також на Дашборді блок «AI-витрати поточного місяця».

### «Як зробити запас у себе?»
Налаштування бренду → AI-фото стиль → є кнопка «Експортувати всі налаштування в JSON» — зберігає всі промпти і поля. У разі чого можна імпортувати назад.

### «Можу я дати другу другому акаунт в адмінці?»
Так. Розділ «Налаштування» → «Користувачі» → «Створити». Налаштуй роль (admin / editor / viewer).

### «Що робити якщо щось зламалось?»
Подивись в `RUNBOOK.md` — там покрокові інструкції для типових ситуацій.
